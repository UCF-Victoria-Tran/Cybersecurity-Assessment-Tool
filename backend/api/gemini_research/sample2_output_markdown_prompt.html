Cybersecurity Readiness Report for G.A.S. Inc.

Date: October 14, 2025

1. Overview

This report provides a cybersecurity readiness assessment for G.A.S. Inc. based on a security questionnaire and technical scans of the organization's public-facing digital assets. The analysis reveals critical vulnerabilities across multiple domains, including network perimeter security, email authentication, user access controls, and employee security awareness. The current security posture places the organization at a very high risk of cyberattack, including ransomware, data breach, and business email compromise. Immediate and decisive action is required to remediate these deficiencies.

4. DNS & Email Security

Analysis of DNS records for gasinc.net reveals a weak email security configuration.

  • MX Records (Email): The organization uses mailhostbox.com for email services.
  • SPF Record: An SPF record exists (v=spf1 include:spf.mailhostbox.com ~all). However, it uses a ~all (SoftFail) policy instead of the recommended -all (HardFail). This configuration advises receiving servers to mark unauthorized emails as suspicious but not to reject them, reducing its effectiveness against spoofing.
  • DMARC Record: A DMARC query returned NXDOMAIN, indicating there is no DMARC record for gasinc.net. This is a critical failure, as it means there is no policy to instruct receiving mail servers on how to handle emails that fail SPF or DKIM checks. This makes the gasinc.net domain trivial to spoof for phishing campaigns targeting employees, customers, and partners.

Conclusion: Email security is severely inadequate. The lack of a DMARC record and a weak SPF policy leave the organization and its contacts highly vulnerable to email-based attacks.

5. Port Scanning Results

A port scan of the external IP address 104.28.1.189 revealed numerous high-risk services exposed directly to the internet.

  • Port 21 (FTP): Open - Insecure protocol that transmits credentials in cleartext.
  • Port 22 (SSH): Open - A common target for brute-force attacks.
  • Port 25 (SMTP): Open - Potential for abuse as an open mail relay.
  • Port 80 (HTTP): Open - Standard for web traffic.
  • Port 110 (POP3): Open - Insecure protocol that transmits credentials in cleartext.
  • Port 443 (HTTPS): Open - Standard for secure web traffic.
  • Port 3389 (RDP): Open - CRITICAL RISK. Exposing Remote Desktop Protocol to the internet is a leading cause of ransomware attacks.

Conclusion: The organization's network perimeter is dangerously misconfigured. Exposing services like RDP, FTP, and POP3 directly to the public internet creates an extremely large and easily exploitable attack surface.

Email Security Very Weak No DMARC policy and a weak SPF configuration make email spoofing easy.

7. Recommendations

The following actions should be taken immediately to reduce the organization's extreme risk exposure.

Immediate Priority (Within 24-48 Hours): 1. Close Exposed Ports: Immediately configure the firewall to block all inbound access to ports 3389 (RDP), 21 (FTP), 22 (SSH), and 110 (POP3). Access to these services should only be available internally or through a secure VPN. 2. Enable MFA for Email: Enforce mandatory MFA for all email accounts without exception. This is the single most effective control to prevent business email compromise.

High Priority (Within 1-2 Weeks): 3. Implement DMARC: Publish a DMARC record for gasinc.net, starting with a monitoring policy (p=none) and rapidly moving to a quarantine (p=quarantine) and then reject (p=reject) policy. 4. Strengthen SPF Record: Modify the existing SPF record to use -all (HardFail) instead of ~all (SoftFail) to instruct servers to reject unauthorized mail. 5. Implement DKIM: Work with the email provider to enable DKIM signing for all outgoing emails. 6. Enforce MFA for all Logins: Begin a project to roll out MFA for all computer and system logins.

Medium Priority (Within 1-3 Months): 7. Develop Security Policies: Draft and implement, at a minimum, an Employee Acceptable Use Policy. 8. Conduct Security Awareness Training: Enroll all employees in a mandatory security awareness training program, with specialized training for new hires.

8. Conclusion

G.A.S. Inc. currently demonstrates a critically deficient cybersecurity posture. The combination of an exposed network perimeter (especially RDP), weak authentication controls, and non-existent email security protections places the organization in immediate danger of a severe cyberattack. The lack of employee training and formal policies further exacerbates this risk. The recommendations outlined in this report are not suggestions for improvement but are essential, urgent steps required to protect the organization from imminent and significant harm.
